Linux não é o sistema operacional

Linux não é o sistema operacional em si, no sentido tradicional. Linux é uma camada de software chamada kernel, que é responsável pela comunicação direta entre os inúmeros hardwares diferentes e o software que vier por cima desse kernel (o resto do sistema operacional e suas aplicações).
Esse kernel foi desenvolvido inicialmente por Linus Torvalds e posteriormente distribuído na licença de código aberto, assim permitindo contribuição da comunidade open-source.

Composição do Kernel

Linux é escrito nas seguintes linguagens de programação e nas respectivas proporções:

  • C: 97%
  • C++, Assembly, Perl: >3%
Em 2021, no total, apenas o kernel já era composto por mais de 30 milhões de linhas de código. Para pra pensar! Deve ter sido no mínimo difícil, especialmente aos meus olhos, já que não consigo sequer escrever um programa decente em Python!

Ideias por trás do Linux como OS

Para facilitar, usaremos a distro Ubuntu para exemplificar o workflow rotineiro do sistema.

Por incrível que pareça, tudo começa nessa parte tão trivial do fluxo de trabalho. Antigamente, por conta dos preços elevados dos computadores, os funcionários de uma empresa utilizavam o mesmo computador (às vezes simultâneamente) no dia-a-dia. Como cada funcionário possui uma função diferente, tipicamente, e cada um tem níveis de privilégio à informação diferentes, fazia-se necessário separar os ambientes de trabalho digitais desses usuários para melhor organização e segurança da informação. A identidade de um usuário no Linux se dá por 2 credenciais principais: Nome de usuário e senha. Para acessar o seu usuário, intuitivamente, basta inserir essas duas credenciais e voilà.
